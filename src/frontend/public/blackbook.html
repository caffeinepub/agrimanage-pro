Abstract

AgriManage Pro is a comprehensive, web-based Agriculture Management System developed to digitize and streamline farm operations for small to medium-scale agricultural businesses in India. The system aims to replace traditional paper-based farm management with a modern, accessible, and efficient digital platform.

The system is built using React 18 and TypeScript for the frontend, and Motoko programming language for the backend, deployed on Internet Computer Protocol (ICP) blockchain. Unlike traditional web applications, no external server or SQL database is required — all data is stored permanently and securely in a decentralized blockchain canister.

The application supports a dual-access model: Regular Users can access the marketplace, notices, weather forecast, livestock information, and place orders without any login. Admin (Farm Owner) has full access to all management modules including farmers, workers, crops, fields, inventory, expenses, tasks, and delivery management through a secure password-protected login.

Key features of the system include:

  • Farmer and Worker Registration & Management with backend persistence
  • Field and Crop Lifecycle Tracking (6 growth stages)
  • Inventory Management with automated Low Stock Alerts
  • Expense Tracking and Monthly Reports & Analytics
  • Online Delivery Marketplace with UPI Payment Integration
  • Automatic Delivery Charge Calculation based on order quantity
  • Notice Board for Farm Communication
  • Weather Forecast and Livestock Management modules
  • Real-time Dashboard with Farm Statistics

The application is fully responsive and accessible on any device — laptop, mobile phone, or tablet — without any installation required. It leverages ICP blockchain infrastructure for permanent, decentralized, and secure data storage without any monthly server costs.

Chapter 1: Introduction

1.1 Overview

AgriManage Pro is a full-stack, web-based Agriculture Management System built to address the real-world challenges faced by farm owners, farmers, and agricultural workers in India. The system provides a unified digital platform covering every aspect of farm management — from crop tracking and worker management to online product delivery with UPI payment integration.

The application is designed to be accessible to all stakeholders without any technical knowledge. Farm owners (admin) can manage all backend operations securely, while regular users such as customers and field workers can access the marketplace, notices, and other user-facing features without any login requirement.

The system is deployed on the Internet Computer Protocol (ICP) blockchain, making it accessible globally via a simple web link on any device without requiring installation. This approach eliminates traditional server costs and ensures permanent, tamper-proof data storage.

1.2 Motivation

India has over 140 million farming households, yet most farm management is done manually using paper registers or simple spreadsheets. Agriculture is the primary source of livelihood for nearly 58% of India's population, yet the sector remains largely undigitized at the farm level.

Key problems that motivated this project include:

  • Loss of important farm records, crop history, and expense data
  • Inability to track farm expenses accurately leading to financial losses
  • No visibility into inventory levels, leading to stockouts or wastage
  • Difficulty coordinating workers and assigning tasks efficiently
  • No direct platform for farmers to sell produce to customers
  • High cost of traditional software systems (monthly subscriptions, servers)

1.3 Problem Statement

Farm owners in India lack a centralized, affordable, and easy-to-use digital system to:

  1. Track crops through all growth stages from planting to harvest
  2. Register and manage farm workers with skill-based assignments
  3. Monitor inventory levels and receive automated low-stock alerts
  4. Record and categorize all farm expenses for financial tracking
  5. Sell produce online directly to customers with integrated payment
  6. Communicate notices and updates to workers and customers

1.4 Proposed Solution

AgriManage Pro provides a complete digital solution by offering:

  • A mobile-friendly web application accessible on any device without installation
  • Secure admin panel for farm owners with comprehensive management tools
  • Public marketplace for customers to browse and order farm produce
  • UPI payment integration with automatic delivery charge calculation
  • ICP blockchain backend ensuring zero server costs and permanent data storage
  • Real-time dashboard with farm statistics and analytics

Chapter 3: Literature Review

3.1 Existing Agriculture Management Systems

Several agriculture management systems have been developed in recent years to address the challenges faced by farmers. However, most existing solutions have significant limitations:

  • AgroStar and DeHaat: Mobile apps focused primarily on advisory services and input supply. They do not provide comprehensive farm management including expense tracking, worker management, or direct customer delivery.
  • iFarmer: Provides farm management features but requires monthly subscription fees and a dedicated server infrastructure, making it expensive for small farmers.
  • Traditional ERP Systems (SAP, Tally): Overly complex and expensive for agricultural use. Require dedicated IT teams and significant hardware investment.
  • Spreadsheet-based Systems: Widely used but prone to data loss, errors, and cannot support online marketplace or real-time collaboration.

3.2 Blockchain in Agriculture

Recent research has highlighted the potential of blockchain technology in agriculture for supply chain transparency, data integrity, and smart contracts. Key findings include:

  • Blockchain ensures tamper-proof record keeping of crop histories and transactions
  • Decentralized storage eliminates single points of failure
  • Smart contracts can automate payment processing when delivery conditions are met
  • ICP blockchain specifically enables hosting complete web applications on-chain

3.3 Research Gap

After reviewing existing systems, the following gaps were identified that AgriManage Pro addresses:

  • No existing affordable system combines farm management + online marketplace + blockchain storage
  • Most systems require installation or monthly server costs
  • Lack of integrated UPI payment with delivery charge management
  • No dual-access model for admin vs public user separation
  • No system fully accessible on all devices without installation

3.4 Proposed Contribution

AgriManage Pro fills these gaps by providing a free, installation-free, blockchain-based agriculture management system with integrated marketplace and UPI payment, accessible to any farmer with a smartphone.

src/backend/main.mo (Part 3 — State & Core Functions)
  // ON-CHAIN STATE (No server or database needed)
  let farmers   = Map.empty<FarmerId, FarmerProfile>();
  let fields    = Map.empty<FieldId, Field>();
  let crops     = Map.empty<CropId, CropRecord>();
  let workers   = Map.empty<WorkerId, WorkerProfile>();
  let tasks     = Map.empty<TaskId, Task>();
  let inventory = Map.empty<ItemId, InventoryItem>();
  let expenses  = Map.empty<ExpenseId, Expense>();
  let notices   = Map.empty<NoticeId, Notice>();

  let accessControlState = AccessControl.initState();
  include MixinAuthorization(accessControlState);

  // FARMER FUNCTIONS
  public shared ({ caller }) func createFarmerProfile(
    profile : FarmerProfile) : async () {
    if (not AccessControl.hasPermission(
          accessControlState, caller, #user)) {
      Runtime.trap("Unauthorized");
    };
    farmers.add(caller, {
      profile with registrationDate = Time.now() });
  };

  public query ({ caller }) func getAllFarmerProfiles()
    : async [FarmerProfile] {
    farmers.values().toArray().sort();
  };

  // WORKER FUNCTIONS
  public shared ({ caller }) func addWorkerProfile(
    profile : WorkerProfile) : async WorkerId {
    if (not AccessControl.isAdmin(accessControlState, caller)) {
      Runtime.trap("Unauthorized: Only admins can add workers");
    };
    workers.add(profile.name,
      { profile with joiningDate = Time.now() });
    profile.name;
  };

  public query ({ caller }) func getAllWorkerProfiles()
    : async [WorkerProfile] {
    workers.values().toArray().sort();
  };

  // TASK FUNCTIONS
  public shared ({ caller }) func createTask(
    task : Task) : async TaskId {
    tasks.add(task.title, { task with status = #pending });
    task.title;
  };

  public shared ({ caller }) func updateTaskStatus(
    taskId : TaskId, status : TaskStatus) : async () {
    switch (tasks.get(taskId)) {
      case (?task) { tasks.add(taskId, { task with status }) };
      case (null) { Runtime.trap("Task does not exist") };
    };
  };

export function useAdminAuth() {
  const [isAuthenticated, setIsAuthenticated] = useState(
    () => sessionStorage.getItem(SESSION_KEY) === "true"
  );

  const login = (password: string): boolean => {
    if (password === ADMIN_PASSWORD) {
      sessionStorage.setItem(SESSION_KEY, "true");
      setIsAuthenticated(true);
      return true;
    }
    return false;  // Wrong password returns false
  };

    sessionStorage.removeItem(SESSION_KEY);

  };

15.1 Overview of On-Chain Storage

Unlike traditional web applications that use relational databases such as MySQL or PostgreSQL, AgriManage Pro stores all data directly on the Internet Computer Protocol (ICP) blockchain. The backend Motoko canister uses in-memory Map data structures that are automatically persisted on the blockchain between calls.

This approach has several advantages: no database server is needed, data is automatically replicated across blockchain nodes, there is no risk of data loss due to server failure, and the storage cost is included in the ICP canister cycles.

15.4 Storage Architecture on ICP

Internet Computer Protocol uses a unique approach to data storage. The canister (smart contract) has its own memory called "stable memory" and "heap memory". In AgriManage Pro:

  • Heap Memory: Fast, in-memory storage for active session data using Motoko Map structures
  • Blockchain Persistence: All canister state is persisted across upgrades using ICP's orthogonal persistence model
  • No External Database: The canister itself acts as the database, eliminating the need for MySQL, MongoDB, or any other database server
  • Replication: ICP automatically replicates canister data across multiple nodes for reliability

15.5 Data Security

Data stored in ICP canisters benefits from blockchain-level security:

  • All state changes are cryptographically verified
  • No single point of failure -- data replicated across nodes
  • Canister upgrades require controller authorization
  • Principal-based access control restricts sensitive operations

Chapter 17: Internet Computer Protocol (ICP) Blockchain

17.1 What is Internet Computer Protocol?

The Internet Computer Protocol (ICP) is a blockchain-based cloud computing platform developed by the DFINITY Foundation. Unlike traditional blockchains that only support financial transactions, ICP can host and run complete web applications including both frontend and backend code directly on the blockchain.

ICP was launched in May 2021 and represents a new vision for the internet: one where applications run on a decentralized network of computers rather than on servers owned by single companies like Amazon, Google, or Microsoft.

17.2 Key Concepts

17.2.1 Canisters

A "canister" is the fundamental unit of deployment on ICP. It is similar to a smart contract on Ethereum, but far more powerful. A canister can:

  • Store gigabytes of data persistently
  • Run complex logic and computations
  • Serve web pages directly to browsers via HTTP
  • Call external APIs via HTTP outcalls
  • Maintain state between calls without a database

17.2.2 Motoko Language

Motoko is a programming language designed specifically for ICP by DFINITY Foundation. It has first-class support for ICP features like:

  • Actor-based programming model (similar to Erlang/Akka)
  • Async/await for inter-canister calls
  • Built-in support for Principal (ICP identity) types
  • Automatic orthogonal persistence (data survives upgrades)
  • Type safety with variant types for enums

17.2.3 Cycles

Canisters consume "cycles" (similar to gas on Ethereum) for computation and storage. Cycles are purchased with ICP tokens. AgriManage Pro consumes very few cycles for its operations, making it extremely cost-effective to run.

Chapter 18: Motoko Programming Language

18.1 Introduction to Motoko

Motoko is a statically typed, modern programming language created by DFINITY Foundation specifically for writing Internet Computer canisters. It was designed to be safe, expressive, and easy to learn for developers familiar with JavaScript or TypeScript.

18.2 Key Language Features

18.2.1 Actor Model

Every Motoko canister is an "actor" -- an isolated unit of computation that communicates with other actors through asynchronous message passing. The actor { } declaration in Motoko maps directly to an ICP canister.

actor {
  // state variables here
  var counter : Nat = 0;

  // public function (accessible via API)
  public func increment() : async Nat {
    counter += 1;
    return counter;
  };

  // query function (read-only, faster)
  public query func getCount() : async Nat {
    return counter;
  };
}

public shared func updateTaskStatus(
  taskId : TaskId,

  switch (tasks.get(taskId)) {
    case (?task) {
      // Task found -- update it
      tasks.add(taskId, { task with status });
    };
    case (null) {
      // Task not found -- trap with descriptive message

  switch (farmers.get(caller)) {

      farmers.add(caller, {

Chapter 36: Blockchain Technology and ICP Deep Dive

36.1 What is Blockchain?

A blockchain is a distributed digital ledger that records transactions across many computers in a way that makes it virtually impossible to alter past records. Unlike a traditional database managed by a single company, a blockchain is maintained by a network of nodes, each holding a copy of the complete data history.

Key properties of blockchain technology used by AgriManage Pro:

  • Immutability: Once data is written, it cannot be silently altered or deleted
  • Decentralization: No single point of failure or control
  • Transparency: All canister operations are verifiable on-chain
  • Permanence: Farm data persists as long as the ICP network operates

36.3 What is a Canister?

A canister is the ICP equivalent of a smart contract. It contains three components:

  • Code (WebAssembly): The compiled Motoko code that defines functions and logic
  • State (Memory): Persistent data stored in the canister's stable memory
  • API: Public functions that can be called by the frontend or other canisters

36.5 ICP Consensus Mechanism: Threshold Relay

ICP uses a proprietary consensus mechanism called Threshold Relay combined with a Byzantine Fault Tolerant (BFT) protocol. This ensures:

  • The network continues operating even if up to one-third of nodes fail or act maliciously
  • Update calls (write operations) achieve finality in approximately 2 seconds
  • Query calls (read operations) are served instantly from a single node without consensus overhead
  • All nodes in a subnet hold identical copies of canister state
